# Route Automation for Keenetic

Автоматизация управления маршрутами OpenVPN на роутерах Keenetic через RCI API. Скрипт получает публичные префиксы из ASN (Meta, Google, Cloudflare), агрегирует их, сравнивает с текущими маршрутами на Keenetic и синхронизирует маршруты, исключая локальные сети. В конце сохраняет конфигурацию на роутере. Можно добавить любые ASN в список.

---

## ⚡ Особенности

- Получение ASN-префиксов через RIPE Stat API  
- Агрегация сетей (`collapse_addresses`) до `/24`  
- Фильтрация локальных и приватных сетей (RFC1918, loopback, link-local, multicast, reserved)  
- Дифф-подход:  
  - Добавляются только недостающие маршруты  
  - Удаляются только лишние публичные маршруты  
- Batch-запросы и логирование прогресса для предотвращения таймаутов  
- Автоматическое сохранение конфигурации на Keenetic  

---

## 📦 Требования

- Python 3.10+  
- Библиотека `requests`

Установка зависимостей:

```bash
pip install requests
```

---

## ⚙ Настройки

В начале скрипта задаются основные параметры:

```python
KEENETIC_URL = "http://10.120.0.1:81/rci/"  # URL RCI API роутера
INTERFACE = "OpenVPN0"                       # Интерфейс для маршрутов
MIN_PREFIX = 24                              # Минимальный префикс при агрегации
BATCH_SIZE = 50                              # Размер batch при отправке на роутер
SLEEP = 0.3                                  # Пауза между batch-запросами
```

Список ASN:

```python
ASNS = {
    "meta": 32934,
    "google": 15169,
    "cloudflare": 13335,
}
```

---

## 🚀 Использование

Запуск скрипта:

```bash
python routes.py
```

Скрипт выполняет следующие шаги:

1. Получение ASN-префиксов  
2. Агрегация и фильтрация публичных сетей  
3. Чтение текущих маршрутов на Keenetic  
4. Сравнение текущих маршрутов с желаемыми (`desired`)  
5. Удаление лишних маршрутов (только публичные)  
6. Добавление новых маршрутов  
7. Сохранение конфигурации на роутере  

В консоль выводятся все изменения и прогресс выполнения.

---

## 🔒 Приватные сети

Скрипт **не трогает локальные сети** (10.x.x.x, 172.16.x.x, 192.168.x.x, loopback, link-local, multicast, reserved).

## 📌 Примечания

- Скрипт не требует авторизации на Keenetic (использует открытый RCI API). Необходимо выполнить проброс порта tcp/79 устройства Кинетик на порт tcp/81 указав ip адрес пк с которого будет запускаться скрипт.  
- Рекомендуется запускать на машине с доступом к роутеру по HTTP RCI.  
- Для больших ASN рекомендуется увеличить `timeout` в функции `keenetic_post`.

---

## ⚙ Настройка для других ASN и интерфейсов

Для добавления новых ASN просто расширьте словарь `ASNS`:

```python
ASNS["new_provider"] = 12345
```

Для изменения интерфейса маршрутизации измените переменную `INTERFACE`.

---

## 🔧 Отладка

Если возникают ошибки таймаута или соединения:

- Проверьте доступность роутера по HTTP `KEENETIC_URL`
- Увеличьте `timeout` в функции `keenetic_post`
- Убедитесь, что машина запуска
